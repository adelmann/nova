Anforderungen

  • PHP 8.1+ mit den Extensions pdo_sqlite, mbstring, dom, gd, zip, curl.
  • Webserver mit mod_rewrite (Apache); Docroot zeigt auf public/.
  • Für die PWA-Installation und Online-Zahlung: HTTPS.

Auf Shared Hosting installieren (empfohlen)

  1. Dateien hochladen

    Das aktuelle Release als ZIP von GitHub laden und inklusive vendor/ in den Webspace entpacken/hochladen. Ein composer install ist nicht nötig. Du brauchst weder .git/ noch die docker-*-Dateien.

  2. Docroot setzen

    Den Docroot der (Sub-)Domain auf den Ordner public/ zeigen lassen. Geht das bei deinem Hoster nicht, lege einfach alles in den Docroot – die mitgelieferte .htaccess leitet nach public/ um und sperrt src/, storage/ und vendor/.

  3. Schreibrechte vergeben

    Stelle sicher, dass der Ordner storage/ beschreibbar ist:

    chmod -R 775 storage

  4. Setup im Browser öffnen

    Rufe deine Domain auf – der Setup-Assistent unter /setup legt das Datenbankschema an, erstellt deinen Admin-Zugang und erfasst die Firmen-Basisdaten.

Alternative per Kommandozeile: 
php bin/migrate.php
php bin/seed-admin.php deine@mail.de "Passwort" "Dein Name"

Lokal entwickeln/testen

Mit Docker

docker compose up --build   # migriert, legt Admin an, läuft auf :8000

Mit nativem PHP

php bin/migrate.php
php -S localhost:8000 -t public public/index.php

Ersteinrichtung

  • Im Setup deine Firmendaten hinterlegen (Name, Anschrift, Steuernummer/USt-IdNr., ggf. Kleinunternehmer-Regelung).
  • Unter Einstellungen → Firma Logo und Briefkopf ergänzen – diese erscheinen auf allen PDFs.
  • Unter Einstellungen → E-Mail den Versand (SMTP oder mail()) sowie Signatur und Textvorlagen einrichten.
  • Bei Bedarf weitere Benutzer einladen und Rollen vergeben (Inhaber, Mitarbeiter, Steuerberater).

Täglicher Ablauf

  1. Kunde & Angebot

    Kunden anlegen, ein Angebot erstellen und – sobald angenommen – per Klick in eine Rechnung umwandeln.

  2. Rechnung festschreiben & versenden

    Die Rechnung erhält beim Festschreiben eine lückenlose Nummer und wird gesperrt (GoBD). Versand als PDF per E-Mail – optional mit Online-Bezahllink.

  3. Zahlungen erfassen

    Zahlungseingänge manuell erfassen oder per CSV-Bankimport automatisch zuordnen. Überfällige Rechnungen lassen sich anmahnen.

  4. Ausgaben & Belege

    Belege als Foto/PDF hochladen und als Ausgabe mit EÜR-Kategorie verbuchen.

  5. Auswerten & exportieren

    Die EÜR jederzeit einsehen und als PDF/CSV, DATEV-Buchungsstapel oder Jahres-ZIP für die Steuerberatung exportieren.

Online-Zahlung einrichten (optional)

Unter Einstellungen → Online-Zahlung. Ohne hinterlegte Schlüssel erscheint kein Bezahllink.

Stripe

  • Im Stripe-Dashboard den Secret Key (sk_live_… bzw. sk_test_…) kopieren und eintragen.
  • Einen Webhook auf die in den Einstellungen angezeigte URL …/webhook/stripe einrichten (Event checkout.session.completed) und das dort erzeugte Signing Secret (whsec_…) hinterlegen.

PayPal

  • Im PayPal-Developer-Dashboard eine App anlegen und Client-ID und Secret eintragen.
  • Modus Sandbox zum Testen, Live für echte Zahlungen. Ein Webhook ist nicht nötig – die Zahlung wird bei der Rückkehr abgeschlossen und verbucht.
Bei beiden Anbietern wird der Bruttobetrag als Einnahme und die Gebühr automatisch als Betriebsausgabe gebucht. Die EÜR-Kategorie für die Gebühr (Standard: Bankgebühren) ist einstellbar.

Backups & Cron

  • Unter Einstellungen → Datensicherung ein Backup auf Knopfdruck erstellen, herunterladen oder löschen.
  • Automatisch per Cron – entweder per CLI:
php bin/backup.php

… oder über den token-geschützten Web-Cron (die genaue URL steht in den Einstellungen):

wget -qO- "https://deine-domain.de/cron/backup?token=…"

Für Wartungsaufgaben (überfällige Rechnungen markieren, automatische Erinnerungen, Update-Check) zusätzlich regelmäßig:

php bin/sweep.php

Updates

Nova prüft GitHub auf neue Releases. Unter Einstellungen → System lässt sich ein Update per Klick installieren: zuerst Backup, dann Download und Entpacken des Release-ZIP, dann die Datenbank-Migrationen. storage/, config.php und die Datenbank bleiben dabei unangetastet.

Als App installieren (PWA)

  • iOS (Safari): Teilen-Menü → „Zum Startbildschirm hinzufügen".
  • Android (Chrome): Menü → „App installieren".
  • Es werden nur statische Assets gecacht – niemals Seiten, PDFs oder Geschäftsdaten.
Wichtig: Nova ersetzt keine Steuerberatung. Prüfe die Auswertungen vor jeder Abgabe und kläre steuerliche Fragen mit deiner Beratung. Nutzung auf eigenes Risiko, ohne Gewährleistung.
